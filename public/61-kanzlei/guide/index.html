How the paragraph became a red thread

Advokatur Steiner & Frey is a practical Swiss SME template — a fictional Basel law firm — built as one self-contained HTML file. No frameworks, no build step: one Google Font, one photograph, one runtime-generated SVG path and about 120 lines of vanilla JS.

The template structure

Hero (photo under a walnut/green scrim, classical wordmark between hairlines, claim, CTA «Beratungstermin») → six Rechtsgebiete cards → Vorgehen in three numbered steps with a Honorar-Transparenz box → three lawyer profiles → Publikationen list → Kontakt with a mock form → footer with a bar-association text badge. All copy is de-CH: no ß anywhere, always «ss», prices in CHF, guillemets («…») for quotes.

Der Paragraph — the signature detail

The Rechtsgebiete grid reserves a narrow left rail for an SVG whose d attribute is built by JS at runtime, so it always fits the measured card layout:

  • The path starts with a §-figure: two stacked, overlapping S-curves drawn as one stroke, closed by a short tail into the vertical axis.
  • From there the red thread (oxblood, 1.7px) descends the full rail, bowing ~14px toward each card row's vertical centre — alternating sides — with a brass node circle at each apex.
  • getTotalLength() sets stroke-dasharray; a passive, rAF-throttled scroll listener maps section progress to stroke-dashoffset, so the line inks itself in as you read. Nodes fill brass once the thread passes them.
  • On resize the whole path is rebuilt (44px rail and a 0.62-scaled § on mobile). With prefers-reduced-motion the thread renders fully drawn and the listener is never attached.

Prägedruck & small-caps discipline

The six cards rest on a faint outset shadow; on hover it flips to a paired inset shadow plus a 1px translateY — the card reads as blind-embossed into the paper rather than lifted off it. Reveals behave like a file folder opening: a slight rotateX(-5deg) from the top edge plus an 18px rise, gated by IntersectionObserver, with no bounce. One serif carries everything — EB Garamond — and all labels, kickers and buttons use font-variant-caps:all-small-caps with wide tracking, which is what gives the page its notarial calm.

The photograph

assets/bibliothek.webp (1920×1072) — a wood-panelled chambers library with leather-bound statute volumes and brass bouillotte lamps — was generated with Nano Banana 2 via the Higgsfield MCP and converted to WebP with ffmpeg. It loads eagerly with fetchpriority="high" and real width/height attributes; everything else on the page is CSS.
