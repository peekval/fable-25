The Guide

How fifty-eight websites
were built from nothing.

This entire showcase — 40 sites, every visual, every animation, all copywriting — was designed and built end-to-end by Fable (Claude) running in Claude Code, autonomously. Wave I shipped 25 dialects; Wave II escalated with 15 heavier 3D pieces. Here is the exact process, so you can run it yourself.

01 — The one rule

No external imagery. Zero. The original plan considered generated images and video-animation pipelines, but those integrations weren't available in the environment — so the constraint became the concept: everything you see is produced by code at runtime. WebGL fragment shaders, three.js scenes, 2D-canvas particle and fluid simulations, generative algorithms with seeded randomness, self-drawing SVG line art, and pure CSS. A constraint like this is worth adopting even when you do have image models: it forces design instead of decoration, keeps every site under ~50 KB, and means nothing on the page is borrowed.

02 — Concept system: forced diversity

25 sites that are "fundamentally different" don't happen by asking for variety — they happen by reserving dimensions before writing a line of code. Each concept locked four things up front, and no two sites were allowed to share more than one:

  • A medium — kinetic type, GLSL shader, 3D scene, terminal, ink canvas, grid system, particles, ornament, light mask, morphing organics, glitch, engraving, weather sim, audio, geometry, metaballs, collage, aurora, isometrics, editorial type, HUD, line drawing, negative space, springs, generative art.
  • A palette — defined as exact hex values, 3–5 colors, with one deliberate accent.
  • A type pairing — two (max three) Google Fonts chosen for the world, not for legibility alone: VT323 for phosphor, Bodoni Moda for couture, Shippori Mincho for Kyoto.
  • A fictional client — real brands force real copywriting. Every site has actual, written content: poems, museum labels, horological specs, case files. No lorem ipsum anywhere.

03 — The fleet

Each site was built by a dedicated agent given a hard specification, in parallel batches of five. The spec that every agent received:

Per-site contract
  • One self-contained index.html — all CSS/JS inline; only Google Fonts (and three.js, once) allowed from a CDN.
  • A complete landing page: hero + at least four real sections + footer, with confident copywriting.
  • Accessibility & performance floors: respect prefers-reduced-motion, cap devicePixelRatio at 2, single rAF loops, pause offscreen canvases, hold up at 390 px and 1440 px.
  • A fixed ← 25 link back to the gallery.

04 — The iteration protocol

No site was accepted after its first draft. Every one went through three mandatory review passes, each required to find and fix at least three concrete flaws:

  1. Pass 1 — structure: missing features vs. spec, spacing rhythm, contrast failures, overflow at mobile widths, z-index collisions.
  2. Pass 2 — motion: easing curves (the difference between "animated" and "alive"), stagger timing, reduced-motion completeness, scroll-trigger thresholds, cleanup of spawned elements.
  3. Pass 3 — elevation: add one detail that lifts the design above competent — a grid toggle, a keyboard shortcut, a breathing hero — plus typography refinement and copy sharpening.

The passes are documented inside each file — open any site and view source: the first line is an HTML comment listing exactly what was fixed in each pass. 25 sites × 3 passes = 75 documented design reviews.

05 — Assembly & deployment

  1. All sites live in one folder — public/<nn-name>/index.html — so the whole show deploys as a single static site.
  2. The gallery hub you arrived from is itself site #26, built last with full knowledge of all 25 palettes (each card's gradient tile uses its site's real colors).
  3. Deployed with netlify deploy --prod --dir public. Total payload for all 26 pages: under one megabyte — smaller than a single hero photograph on most websites.

06 — Reproduce it

Do this yourself, with any agentic coding tool
  1. Write the concept table first — media, palettes, type pairs, fictional clients. Diversity is decided here, not in the prompts.
  2. Adopt the code-only asset rule. It is the single highest-leverage constraint.
  3. Give each build a contract, not a vibe: exact hexes, exact fonts, named techniques, performance floors, reduced-motion.
  4. Run builds in parallel, five at a time.
  5. Enforce three adversarial review passes with a minimum fix count — and make the builder document them in the artifact itself.
  6. Ship everything as one static deploy with a gallery hub, so the work is browsable as a body of work.

07 — Wave II: escalation & reproduction prompts

The 15 Wave II sites (26–40) push into heavier real-time work — instanced voxel assembly, raymarched fractals, Verlet cloth, fluid dye advection, procedural planets, software ASCII rendering, holographic foil — under a strict performance contract: 60fps targets, instancing over per-object meshes, reduced internal shader resolution, capped particle counts with mobile fallbacks, paused offscreen renderers.

Every site — all 40, both waves — carries its own /guide route (linked from its footer). Each one documents how that site was built and includes a boxed, copy-paste reproduction prompt — the exact palette hexes, font names, signature technique, layout principle and mood, precise enough for any capable AI to rebuild a similar site, without disclosing a line of source code. That is the template: sell the recipe, keep the craft.
