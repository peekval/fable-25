MAKING-OF.TXT1 OF 1NO CODE INSIDE

The tube is real now: the whole zine renders through a raw-WebGL CRT pipeline. The live page — boot sequence, typing poems, reader console — is mirrored onto an offscreen 2D canvas — each glyph run measured off the actual DOM with Range rects so layout stays pixel-true, re-rastered only when the page actually changes (typing, scroll, a cursor blink), with the static glyphs cached between changes — then post-processed: a ping-pong phosphor-persistence buffer makes bright glyphs leave decaying green trails, a threshold-and-blur bloom chain makes headlines bleed light, and a composite shader bends everything through barrel-curved glass with an RGB aperture grille, rolling scanlines, edge chromatic fringe, vignette, grain, flicker, the occasional horizontal sync wobble, a degauss ripple, and a power-on that collapses the raster to one bright line before blooming open.

Because the DOM keeps doing the work underneath, everything stays functional: pointer input is forwarded through the inverse barrel map so links and the console remain clickable through the curved glass. On arrival a boot sequence types itself out, complete with a counting 64K memory test and an unstable dream module, skippable with any key. Each poem types over an invisible ghost copy on scroll, cursor blinking, pausing at punctuation. The reader console gained tube hardware: `crt` exposes brightness, curvature and scanline knobs, and `degauss` thumps the field clean.

// no WebGL → the pure-CSS scanline tube takes over; reduced-motion gets one static frame; the beam runs at ~30fps with the whole internal pipeline at ~0.6× resolution (CRTs are soft — the glass hides it), DPR capped at 1.25, and the loop sleeps when the tab hides.
