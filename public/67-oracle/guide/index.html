ORACLE — It has been expecting you

How this experience works — and the prompt that rebuilds it. All of it is code: no image or audio assets, everything procedural.

01 The presence

A living eye rendered from flowing radial filaments on canvas: it tracks the cursor precisely, the pupil dilates as you approach, it blinks naturally (rare double blinks), looks around idly when you stop, and squints on fast scroll.

02 The behavior engine

The page quietly observes ~10 behaviors — idling, rushing, circling the eye, leaving and returning to the tab, selecting text, a second visit (localStorage), nearing the end — and answers each exactly once per session with a typeset line ('You stopped. So did it.'). No nagging: every line fires once, perfectly timed.

03 The deterministic oracle

One input: ask it anything. The question hashes to one of ~40 crafted delphic answers — and the same question always returns the same answer. The copy dares you to test it: 'Ask again. It does not change its mind.'

04 Fallbacks

Reduced motion stills the filaments but keeps tracking and the dialogue; localStorage is guarded for file:// contexts; the loop pauses offscreen and on hidden tabs.

Reproduction prompt
Build a single-file uncanny-beautiful experience called "ORACLE — It has been expecting you." Near-black (#0B0C09) with one luminous gold-green accent (#C9CFA0), a serene classical serif, vast negative space. Hero: a living eye built from flowing radial canvas filaments — precise cursor tracking, pupil dilation with cursor proximity, natural blinking with rare double-blinks, idle wandering gaze, squinting under fast scroll; on touch devices it follows touch and scroll. Beneath it, a caption line types out behavior-reactions, each exactly once per session: idle 10s ("You stopped. So did it."), rapid scrolling ("No need to rush."), circling the eye ("It noticed."), tab return via visibilitychange ("Welcome back."), text selection ("Take it. It was written for you."), second visit via guarded localStorage ("It remembers you."), reaching the end ("You want to know how it ends."). Keep a "What it sees" section listing the mentions as they occur. Then "The dialogue": one input field; hash the question deterministically into one of ~40 hand-written delphic answers revealed slowly — same question, same answer, forever; say so in copy. Close with a deadpan about-section: "ORACLE is a page. Pages do not see. Probably." prefers-reduced-motion: becalmed eye, everything else functional. Pause loops offscreen/hidden; no horizontal overflow at 390px; zero console errors.
