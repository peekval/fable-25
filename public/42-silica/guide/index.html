How SILICA was built

One HTML file, no libraries, no WebGL. The whole chip is five flat SVG planes in a CSS preserve-3d isometric stack — assembled by scroll, lit by dash-offset math.

The exploded-die scrub

The hero is a 430vh section with a sticky 100vh viewport. Each die layer (substrate, compute tiles, interconnect mesh, SRAM banks, heat spreader) is an SVG on an absolutely-positioned plane inside a transform-style: preserve-3d container rotated rotateX(60deg) rotateZ(45deg) — with no perspective, so the projection is orthographic and every edge stays parallel and crisp.

Scroll progress maps to five staggered per-layer windows. The trick that makes assembly feel mechanical: gap collapse is cumulative bottom-up — layer i sits at z = Σ (T + GAP·(1−e_j)), so when a lower gap seats, the entire upper stack rides down with it. Each window eases with a mild easeOutBack for a snap-and-settle landing, and a label with a leader line flies in as its layer seats. The stage counter-translates by 0.4 × totalZ so the shrinking stack stays optically centered. The scrub itself is a dirty-flag rAF: scroll events only set a flag, one frame applies it, and nothing runs while idle.

Trace pulses & the x-ray state

At 90% progress the die goes “live”: the heat spreader and SRAM fade to ghost opacity (an x-ray view, so the mesh stays visible), and pulses stream along the interconnect. Every mesh edge is drawn twice — a dim base path and a pathLength="100" pulse copy with stroke-dasharray: 9 91 animated on stroke-dashoffset, with per-edge duration/negative-delay jitter so traffic never looks looped. Hovering a compute tile adds a .hot class to its adjacent edges (a tile→edges map built while generating the SVG) and lights its local traffic lime. All 144 substrate pads, 16 tiles, 24 edges and 6 SRAM banks are generated in ~40 lines of JS instead of hand-written markup.

Lifecycle hygiene: an IntersectionObserver on the sticky viewport plus a visibilitychange listener toggle animation-play-state: paused on every pulse when the hero is offscreen or the tab is hidden. prefers-reduced-motion collapses the hero to a single static assembled viewport with counters landed and static dashes.

The die shot (asset pipeline)

The macro photo of the iridescent die was generated with Nano Banana 2 via the Higgsfield MCP at 2K, palette-locked to the site (violet #8B7CF6 / cyan #5EEAD4 iridescence on near-black, one lime pad dead center), then encoded to WebP with ffmpeg (1920×1080, ~335 KB). It's revealed by a wafer-scan: a lime scanline sweeps down while the image’s clip-path: inset() opens behind it, then an engraved mono caption fades up. The img is lazy-loaded with real width/height attributes.

Everything else

  • Fonts: Familjen Grotesk (technical grotesque display) + Spline Sans Mono — deliberately not Inter/Space Grotesk.
  • Architecture story: a sticky mini-stack of five CSS plates; IntersectionObserver picks the step in the middle band of the viewport and lifts that plate with a lime outline.
  • Benchmarks: bars are transform: scaleX(var(--w)) from origin-left; numbers count up once with cubic ease and tabular-nums.
  • Mobile: die size clamped so the iso diagonal fits 390px; side labels swap for a single mono readout line under the die.
