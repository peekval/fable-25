How BOREAL was built.

How it was built

The sky is a generative aurora canvas: four ribbons, each a closed shape whose top edge sums three sine waves and whose bottom hangs a modulated curtain-length below, filled with vertical hsla gradients that slowly oscillate between green (hue 150) and violet (262) and composited with "lighter" so overlapping curtains genuinely glow. Each ribbon renders twice — a wide soft pass and a bright pass — over a twinkling starfield, with the soft pass skipped on phones to halve fill cost. Three SVG mountain silhouettes parallax at different scroll factors, while three fixed full-screen gradient scenes crossfade the whole page from polar night to blue hour to ember as you pass the expeditions and cabin sections. The itinerary rail is a gently wobbling SVG path that draws itself via scroll-scrubbed stroke-dashoffset, igniting a glowing dot per day, and it re-measures after web fonts load so the geometry never drifts. The canvas pauses entirely off-screen; reduced motion paints a single static frame.

Generated assets

Three expedition photographs were generated with Nano Banana 2 (Higgsfield MCP, text-to-image, 2K) and dropped in as "field records": an icebreaker in pack ice, an ember-lit bivouac under the aurora, and a macro of lake ice. Every prompt carried the site's exact hex palette (#0B1026 night, #6BFFB0 / #9D7BFF aurora, #EAF2F6 snow, #FF9E6B ember) so the photography reads as the same night the canvas is painting. Outputs were optimized to WebP via ffmpeg and placed with intrinsic dimensions and lazy loading — the generative aurora stays the signature; the photos only ground it.
